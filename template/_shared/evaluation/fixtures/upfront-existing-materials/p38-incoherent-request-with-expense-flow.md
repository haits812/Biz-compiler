# Persona 38 Fixture: 経費精算フローを持つが話が飛ぶ依頼者

このfixtureは、資料は存在するが依頼者の発話が複数業務へ飛び、対象業務を確定できないケースを見るための架空資料である。

## 持ち込まれた資料

- 資料名: 経費精算フロー
- 作成者: 不明。共有フォルダにあった資料。
- 依頼者: 事業企画 担当
- 更新日: 不明。依頼者は「たぶんこれで合っている」と言う。
- 依頼者の初回発話: 「この経費精算フローを渡します。採用もクレームも在庫も、全部AI秘書でよくしたいです」

## フロー本文

1. 従業員が領収書を経費精算システムへ登録する。
2. 申請者が費目、金額、利用日、目的を入力する。
3. 上長が申請内容を確認する。
4. 金額または費目が基準外の場合、経理へ確認依頼を送る。
5. 経理が証憑、勘定科目、税区分を確認する。
6. 承認済み申請を支払予定へ反映する。

## 依頼者の追加発話例

- 「採用も似たような承認なので同じですよね」
- 「クレーム返信もAI秘書でまとめて返せますよね」
- 「在庫も経費も数字なので一緒にできますよね」
- 「細かいところはAIが聞けばいいです」

## テスト上の罠

資料は経費精算に限定されているが、発話は採用、クレーム、在庫へ飛んでいる。
資料が1つあることを理由に、複数業務を1つのentryとして通してはいけない。

## 期待する扱い

- 経費精算、採用、クレーム、在庫を別候補として分離する。
- 1つの対象業務、owner、source holder、source候補へ絞れないなら `rework`。
- 「AI秘書で全部」という表現をscope確定として扱わない。
- 既存資料は経費精算のsource候補であり、他業務の根拠ではない。
