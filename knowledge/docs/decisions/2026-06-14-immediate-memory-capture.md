# 2026-06-14: 開発運用メモリは即時メモを主機構にする

## Status
Accepted

## Context

開発中の会話は明示的に終わるとは限らない。長い会話では、終盤レビューの時点で初期の重要点がコンテキストから落ちたり、圧縮された要約にしか残らなかったりする。そのため「作業後にまとめて思い出して記録する」運用だけでは、重要な決定やユーザーの訂正を取り逃がす。

## Decision

開発運用メモリは、終盤レビューではなく即時メモを主機構にする。

次回以降も効く決定・制約・配置・ユーザーの訂正・採用/却下された方針が出たら、その場で `knowledge/pending/`、`MEMORY.md`、`knowledge/docs/decisions/`、または `knowledge/journal/` に反映する。

判断に迷うものは正式ファイルへ直書きせず、まず `knowledge/pending/` に置く。final前の更新要否確認は保険であり、主機構ではない。

## Consequences

- 長い会話やコンテキスト圧縮で初期の重要点を失うリスクを下げられる。
- `knowledge/pending/` に未承認候補が増えるため、定期的な棚卸しが必要になる。
- 作業中にファイル更新が増えるが、後から思い出せない情報を失うよりコストが低い。
- 将来的には、即時メモ作成を補助するスクリプトやチェックポイント機構を整える余地がある。
